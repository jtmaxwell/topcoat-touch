Initialization

The TopcoatTouch is instantiated by calling the TopcoatTouch function. var tt = new TopcoatTouch([container], [options]); Where container is query object (we will use the generic term query object for jQuery or Zepto query results) that is the base container where you wish the TopcoatTouch environment to be initialized. Almost always this is the $('body'), which is the default if no container is passed, but it can be any container (for testing, for example).

Configurable options

exceptionOnError
default false
Whether or not an exception is thrown if an error occurs. By default only a console error is emitted
initializeFunction
default none
Override the default template initialization function in the MVC version of TopcoatTouch. Both the initializeFunction and the renderFunction all called within the context of a PageController. For example, you can choose to use Handlebars as a templating engine by overriding the initialization function: Both initializeFunction and renderFunction all called within the var tt = new TopcoatTouch({initializeFunction: function() { // Need access to the pageController inside of the jquery callback var self = this; // this.tt is available in the initializeFunction and the renderFunction $.get(this.tt.options.templateDirectory + '/' + this.pageName + '.handlebars', function (source) { self.template = Handlebars.compile(source); }); }});
renderFunction
default none
The render function can also be overriden from the default lodash templating. For example to use Mustache as a templating engine: var tt = new TopcoatTouch({initializeFunction: function() { var self = this; $.get(this.tt.options.templateDirectory + '/' + this.pageName + '.mustache', function (source) { self.template = source; }); }, renderFunction: function() { return Mustache.render(this.template, this.data); } });
menu
default false
Set to true if you want to set the menu on each page, or add items as a JSON array that will appear on every page. Eg: [{id: 'help', name: 'Help'}, {id: 'info', name: 'Info'}, {id: 'about', name: 'About'}]
menuFadeIn
default 100
Speed to fade in the menu.
menuFadeOut
default 50
Speed to fade out the menu.
menuHasIcons
default false
Whether or not the menu has icons, if set to true a span with an menuItemIcon class is added to each menu item. in the CSS you should change the icon with #menuItemId .menuItemIcon selector. e.g: #menuAbout .menuItemIcon { background-image: url(images/menuAbout.png); }
templateDirectory
default 'templates'
